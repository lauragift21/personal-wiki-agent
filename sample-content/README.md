# Sample Knowledge Base Content

This directory contains sample documents you can upload to your Personal Wiki Agent to test its search and knowledge management capabilities.

## Documents Included

### 1. **software-architecture.md**

Comprehensive guide covering:

- Software architecture principles (SOLID, loose coupling, cohesion)
- Architectural patterns (Microservices, Monolith, Event-Driven, Serverless)
- Scalability patterns (Horizontal/Vertical scaling, Database strategies)
- Design patterns (Creational, Structural, Behavioral)
- System design considerations (CAP Theorem, Load Balancing, Rate Limiting)

**Search for:** microservices, scalability patterns, design patterns, system design, CAP theorem

### 2. **cloudflare-workers-guide.md**

Complete guide to Cloudflare Workers:

- Core concepts (Edge network, Request lifecycle)

- Workers AI integration
- Advanced features (Service bindings, Queues, Cron triggers)
- Security best practices

**Search for:** workers, kv storage, durable objects, edge computing, serverless

### 3. **typescript-best-practices.md**

## How to Use

1. **Upload individual documents:**
   Use the chat interface to upload these markdown files one by one.

2. **Test search capabilities:**
   After uploading, try searching for topics like:
   - "microservices best practices"
   - "react hooks patterns"
   - "database indexing strategies"
   - "kubernetes deployment"
   - "typescript generics"

3. **Compare retrieval methods:**
   Use the `/compare` command to see differences between:
   - Vector search (semantic similarity)
   - Keyword search (exact matching)
   - Hybrid search (combined)

4. **Ask questions:**
   The AI agent will search your wiki and cite sources, e.g.:
   - "How do I design a URL shortener?"
   - "What are the best practices for React state management?"
   - "Explain the CAP theorem"
